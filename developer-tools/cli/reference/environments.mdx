title: Environments

<Tip>
    If you set a default project in your config you can omit the `--project-id` flag from the commands below.

    ```
    launch config set-default-project project_XXXXX
    ```
</Tip>

## Clear Env Vars

```
                                                                                                                                        
 Usage: launch environments clear-env-vars [OPTIONS]                                                                                    
                                                                                                                                        
 Clear environment variables in an environment.                                                                                         
                                                                                                                                        
╭─ Options ────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────╮
│    --project-id                         TEXT  The id of the project to apply the operation on. If this is unset we will use the      │
│                                               default project set in the config. Should be of the format:`project_XXXXX`             │
│                                               [default: None]                                                                        │
│ *  --name                               TEXT  Name of the environment. [default: None] [required]                                    │
│    --environment-variable-name  -e      TEXT  Environment variable names that should be removed from the environment. Multiple can   │
│                                               be provided with -e NAME1 -e NAME2                                                     │
│                                               [default: None]                                                                        │
│    --help                                     Show this message and exit.                                                            │
╰──────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────╯
```

## Create

```
                                                                                                                                        
 Usage: launch environments create [OPTIONS]                                                                                            
                                                                                                                                        
 Create an environment in a project                                                                                                     
                                                                                                                                        
╭─ Options ────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────╮
│    --project-id                    TEXT         The id of the project to apply the operation on. If this is unset we will use the    │
│                                                 default project set in the config. Should be of the format:`project_XXXXX`           │
│                                                 [default: None]                                                                      │
│ *  --name                          TEXT         Name of the environment this must be unique within the project [default: None]       │
│                                                 [required]                                                                           │
│    --environment-variable  -e      FROM_STRING  A map of environment variables to set on the environment. Should be of the format    │
│                                                 --environment-variable NAME=VALUE --environment-variable NAME2=VALUE2. NOTE: you can │
│                                                 use -e for shorthand (e.g. -e NAME=VALUE -e NAME2=VALUE2))                           │
│                                                 [default: None]                                                                      │
│    --help                                       Show this message and exit.                                                          │
╰──────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────╯
```

## Update

```
                                                                                                                                        
 Usage: launch environments update [OPTIONS]                                                                                            
                                                                                                                                        
 Update an environment.                                                                                                                 
                                                                                                                                        
╭─ Options ────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────╮
│    --project-id         TEXT     The id of the project to apply the operation on. If this is unset we will use the default project   │
│                                  set in the config. Should be of the format:`project_XXXXX`                                          │
│                                  [default: None]                                                                                     │
│ *  --name               TEXT     Name of the environment this must be unique within the project [default: None] [required]           │
│    --num-cpus           INTEGER  Number of CPUs to allocate to each worker in the environment. If not set this will not be updated   │
│                                  in the environment.                                                                                 │
│                                  [default: None]                                                                                     │
│    --max-workers        INTEGER  Maximum number of workers a deployment is allowed to scale up to. If not set this will not be       │
│                                  updated in the environment.                                                                         │
│                                  [default: None]                                                                                     │
│    --help                        Show this message and exit.                                                                         │
╰──────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────╯
```
